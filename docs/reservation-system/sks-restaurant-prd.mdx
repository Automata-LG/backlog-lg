## 1. 제품 비전 및 문제 정의

### 1.1 비전 스테이트먼트

<Card title="제품 비전" icon="bullseye">
  **"LG 멤버십 고객에게 프리미엄 다이닝 경험을 제공하는 스마트 예약 플랫폼"**

  SKS 서울 레스토랑 예약 시스템은 LG 멤버십 회원들이 쉽고 편리하게 프리미엄 레스토랑을 예약하고,
  등급에 따른 차별화된 혜택을 누릴 수 있는 디지털 예약 서비스입니다.
</Card>

### 1.2 문제 정의

<CardGroup cols={2}>
  <Card title="현재 문제점" icon="circle-xmark" color="#ef4444">
    - 전화/오프라인 예약으로 인한 운영 부담
    - 멤버십 등급별 혜택 차별화 부재
    - 수동 테이블 관리로 인한 오류
    - 취소 시 효율적 재배정 시스템 부재
  </Card>
  <Card title="해결 방향" icon="circle-check" color="#10b981">
    - 디지털 예약 시스템 자동화
    - 등급별 예약 가능 기간 차별화
    - 체계적인 테이블 관리 시스템
    - 실시간 대기자 알림 시스템
  </Card>
</CardGroup>

### 1.3 시스템 구조

**Host 구분**
- 3개 Host 운영: LG 아트센터, SKS 다이닝, SKS 카페
- 앱에서 "SKS 레스토랑"으로 통합 표시 (앱 하드코딩)
- 각 Host는 독립적인 Content 운영

### 1.4 가치 제안

| 대상 | 핵심 가치 |
|------|-----------|
| **VIP 고객** | 2개월 전 예약 가능한 독점적 접근권 |
| **일반 회원** | 편리한 모바일 예약 및 쿠폰 할인 혜택 |
| **레스토랑 운영팀** | 효율적인 예약/테이블 관리 및 노쇼 감소 |

## 3. 기능 요구사항

### 3.1 예약 정책 및 입력 데이터

#### FR-001: 예약 오픈 정책
**등급별 예약 기간**
- 블랙: 매월 1일 10시 → 익월, 익익월
- 블랙 미만: 매월 1일 10시 → 익월

**우선순위**: P0 (필수)

#### FR-002: 예약 제한 정책
**예약 제한**
- 1인당 1일 컨텐츠당 1건 (다이닝, 카페 각각)
- 당일 예약 불가 (전일 23:59:59까지만 가능)
- 홀 타입 5-6인 예약: 하루 최대 2건 (6인 테이블 2개 제한)
- 회원식별자 기준 중복 예약 방지
- 타임슬롯 없이 일 단위 정원제 운영 (정원 차면 예약 마감)

**예약 변경/취소**
- 유저 취소: 이용일 2일 전 23:59:59까지
- 유저 변경: 이용일 2일 전 23:59:59까지
- 관리자 취소/변경: 이용 시간 직전까지
- 수정: 테이블 배정 전후로 다름 (FR-005 참조)

**우선순위**: P0 (필수)

#### FR-003: 예약 플로우 및 입력 데이터

**예약 플로우 Step**
- Step 0: 이용안내 및 유의사항 (Content.instructions)
- Step 1: 일정/좌석 선택
- Step 2: 인원/시간 선택 & 가격 확인
- Step 3: 요청사항 입력
- Step 4: 확인 및 약관 동의

**기본 정보**
- 이름: 회원 정보에서 불러오기 (수정은 LMP에서)
- 전화번호: 회원 정보에서 불러오기 (수정은 LMP에서)
- 예약일자: 예약 가능 기간 내 선택
- 좌석 타입: 룸/홀 단일 선택
- 인원:
  - 룸: 4-8인
  - 홀: 2-4인 또는 2-6인
- 방문시간: 30분 단위
  - 다이닝: 11:00-14:00
  - 카페: 15:00-18:00

**추가 정보**
- 요청사항: 텍스트 입력 (최대 100자)
  - 플레이스홀더: "알러지, 유아의자 필요 등"
- 추가 옵션: 복수 선택 가능
  - 쇼룸 투어
  - 제품 상담
  - 각 옵션별 한줄 설명 또는 아이콘 표시
- 차량 정보:
  - 차량 이용 여부 체크박스
  - 차량 번호 입력 (기존 등록 번호 자동 불러오기)
  - 수정해도 등록된 차량번호에는 영향 없음

**우선순위**: P0 (필수)

### 3.2 테이블 관리

#### FR-004: 섹션 구성
**섹션 정의**
- 프라이빗: 룸 1개 (최대 8인)
- 오픈: 홀 8개 테이블
  - 4인 테이블: 6개
  - 6인 테이블: 2개
- 구조: 섹션(ReservationSection) → 테이블(ReservationUnit)

**우선순위**: P0 (필수)

#### FR-005: 테이블 배정
**배정 권한**
- 관리자 수동 배정
- 관리자 재배정 가능
- 룸 선택 시 자동 배정 (룸 1개뿐)

**배정 규칙**
- 크로스 배정 불가 (룸↔홀 불가능)
- 홀 내에서 4인↔6인 테이블 재배정 가능
- 유저는 4인/6인 테이블 구분 불필요 (관리자만 구분)

**수정 권한**
- 배정 전:
  - 날짜/시간 수정 가능
  - 인원 변경 불가
- 배정 후:
  - 인원 변경 (좌석 배정 상태에 따라 제한)
    - 좌석 배정 전: 변경 불가
    - 좌석 배정 후:
      - 4인 테이블: 2-4인 범위 내 변경
      - 6인 테이블: 2-6인 범위 내 변경
  - 방문시간, 요청사항, 차량 유무 및 번호, 추가 옵션 변경 가능

**우선순위**: P0 (필수)

### 3.3 빈자리 알림

#### FR-006: 알림 신청
**신청 조건**
- 만석 시 신청 가능
- 하루 최대 10명 제한
- 날짜 단위 신청 (타임슬롯 아님)
- 알림 받고 직접 예약 필요 (대기 자동 전환 아님)

**우선순위**: P1 (중요)

#### FR-007: 알림 발송
**발송 방식**
- 취소 발생 시 신청자 전체에게 동시 알림
- 선착순 예약 경쟁

**우선순위**: P1 (중요)

### 3.4 결제 및 체크인

#### FR-008: 쿠폰 관리
**쿠폰 종류**
- 다이닝: 식사 할인권 (50%/30%/20%/10% 등급별)
- 카페: 시그니처 음료 제공권

**일반 예약**
- 현장에서 컨텐츠별 쿠폰 차감
- 예약 시점에는 검증만
- 예상 청구 금액 표시 (쿠폰 차감 후)
- 취소 시 쿠폰은 사용 전이므로 별도 복구 불필요

**관리자 예약**
- 쿠폰 차감 없음

**우선순위**: P0 (필수)

#### FR-009: 현장 체크인
**체크인 프로세스**
- 관리자 확인
- 예약 검색
- 이용 완료 처리

**우선순위**: P0 (필수)

### 3.5 알림 시스템

#### FR-010: 통합 알림
**채널**
- 카카오 알림톡
- 앱 푸시

**시점별 알림**
- 예약: 확정/변경/취소
- 오픈: 예약 오픈
- 대기: 빈자리 발생
- 리마인더: D-1, D-day

**우선순위**: P1 (중요)

### 3.6 컨텐츠 관리

#### FR-011: 레스토랑 정보
- 위치, 운영시간, 연락처
- 메뉴 정보 및 사진
- **우선순위**: P0 (필수)

#### FR-012: 소식 게시판
- 공지사항/이벤트 안내
- 관리자 작성/수정/삭제
- **우선순위**: P2 (낮음)

### 3.7 부가 기능

#### FR-013: 초대장 및 공유
**초대장 보내기**
- 예약 완료 후 초대장 보내기 기능
- 카카오톡 링크 생성
- 예약일자, 시간, 장소 정보 공유

**우선순위**: P2 (낮음)

### 3.8 관리자 기능

#### FR-014: 관리자 예약
**특수 예약**
- 회원 조회 없이 직접 입력
- 쿠폰 차감 없음
- 정보 직접 입력:
  - 이름: 텍스트 직접 입력
  - 전화번호: 텍스트 직접 입력
  - 예약일자: 예약 가능 기간 내 선택
  - 좌석 타입: 룸/홀 선택
  - 인원: 최소 2인, 최대 룸 8인/홀 4-6인 (룸 최소 4인)
  - 방문시간: 30분 단위
  - 알러지: 텍스트 입력 (최대 100자)
  - 추가 옵션: 쇼룸 투어, 제품상담

**강제 관리**
- 생성/변경/취소: 이용 시간 직전까지 가능
- 예약 가능 기간은 영향 받음

**우선순위**: P0 (필수)
